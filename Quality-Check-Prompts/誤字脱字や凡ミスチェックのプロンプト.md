# 誤字脱字や凡ミスチェックのプロンプト

タイプ: プロンプト

## 誤字脱字とリンク切れのチェック

```
サイト全体を確認して、以下の3点をチェックしてください。

①誤字脱字とリンク切れがないか
②すべてのリンクが適切にエスケープ処理されているか
③画像や遷移先などで開発環境のローカルパスが残っていないか。 
```

## 英字の大文字の仕様確認

```
サイト全体を確認して、大文字と小文字の表記方法がすべて統一されているか確認して。
（CONTACT→Contact など text-transform: uppercaseで対応)
```

## 不要なコメントの確認

```
サイト全体を確認して、必要ないコメント（仕様の解説ではなくTODO: など）が残っていないかチェックしてください。
```

## HTMLの予約済み文字仕様の確認

```
サイト全体を確認して、HTMLの予約済み文字またはそれに類する扱いの文字を文字参照しているかどうか確認して(現時点では以下のみ)
(< (→ &lt; )
> (→ &gt; )
& (→ &amp; )
" (→ &quot; )
' (→ &apos; ))
```

## timeタグ・date-time属性の利用

```
サイト全体を確認して、日付や時刻を表す箇所は以下の2つが使われているか確認して。 
①timeタグ
②date-time属性
```

## 画像の属性

```
サイト全体を確認して、画像には以下の3つが適切に行われている・設定されているか確認して。 

①ファーストビュー以外にはLoding="lazy"がかかっているか。
②レイアウトシフト対策（Width,Height もしくは Aspect Ratio の指定）が行われているか。 
③静的な画像は基本的にWebpになっているか。
```

## クラス名の統一性

```
クラス名の統一性には矛盾がないか、コードベース全体から徹底的にチェックして。
camelCaseやBEMなど案件によります。 
```

## カスタムフィールド未入力時のPHPエラー

```
ワードプレス環境下で、カスタムフィールドが未入力（nullまたは空文字列）の場合にPHPエラーが発生しないか確認したいです。
以下の点に注目して、コードブロック全体をレビューしてください。

1.	カスタムフィールドの取得箇所: ⁠get_post_meta() や ACF などのカスタムフィールド関数で値を取得している全箇所。

2.	値の存在チェック: 取得した値を使用する前に、⁠empty() や ⁠isset() などのPHP関数を用いて、値の有無や空でないかを適切にチェックしているか。

3.	エラーハンドリング: チェックの結果、値がない場合に適切なデフォルト値の設定や代替処理が記述されているか。
特に、カスタムフィールドの値を直接出力したり、計算に使用したりする箇所は重点的に確認をお願いします！
```
