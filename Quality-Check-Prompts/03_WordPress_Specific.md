# WordPress 特有の実装 チェック用プロンプト

タイプ: プロンプト

## WordPress テーマ開発の基本チェック

```
WordPressテーマファイル（PHP）の実装について確認してください。

1. header.php に `wp_head();`、footer.php に `wp_footer();` が記述されているか（プラグインや管理画面バーの動作に必須）
2. テンプレートファイル内で `WP_Query` や `get_posts` を使用した後、適切に `wp_reset_postdata();` または `wp_reset_query();` が行われているか
3. `target="_blank"` を使用するリンクが動的に生成される場合、セキュリティ対策が含まれているか
4. 投稿サムネイル（アイキャッチ画像）の出力時に、alt属性が適切に出力されるコードになっているか
```

## カスタムフィールドとエラーハンドリング

```
カスタムフィールド（ACF等）の取得と表示処理について、重点的に確認してください。

1. カスタムフィールドの値を取得する際（`get_field`, `get_post_meta` 等）、値が空（null, false, 空文字）の場合の処理が記述されているか
2. 未入力時にPHPエラー（Trying to access array offset on value of type null 等）が発生しないよう、`if`文や `empty()`, `isset()` で存在確認を行っているか
3. カスタムフィールドの値を `echo` する際、適切なエスケープ関数（`esc_html`, `esc_url` 等）を通しているか
```

## functions.php と設定コードのチェック

```
functions.php の設定内容について確認してください。

1. 画像アップロード時に不要なサイズが大量に生成されないよう、自動生成サイズの制御（`remove_image_size` やオプション設定の無効化）コードが含まれているか（要件による）
2. セキュリティやパフォーマンスに悪影響を与える不要なヘッダー情報（WordPressバージョンの出力など）を削除するコード記述があるか
```
