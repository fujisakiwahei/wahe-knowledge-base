# CSS・デザイン・レスポンシブ実装 チェック用プロンプト

タイプ: プロンプト

## 画像とパフォーマンスの実装チェック

```
画像の読み込みと最適化について、コードを確認してください。

1. ファーストビュー（FV）以外の画像に loading="lazy" 属性が付与されているか
2. レイアウトシフト（CLS）を防ぐため、画像に width / height 属性、またはCSSで aspect-ratio が指定されているか
3. 静的な画像に対して WebP 形式が使用されているか（pictureタグでの出し分け含む）
4. Retinaディスプレイ対応（2倍サイズ画像の表示など）が考慮されているか
5. SP表示とPC表示で画像が異なる場合、pictureタグなどを用いて適切に出し分けられているか
```

## CSS・スタイルの記述ルールチェック

```
CSS（またはSCSS）の記述について、品質とルールを確認してください。

1. クラス名の命名規則に一貫性があるか（BEM記法、camelCaseなど、プロジェクトのルールに準拠しているか）
2. HTMLタグに直接 `style="..."` を書くインラインスタイルが使用されていないか（JSによる動的付与を除く）
3. 余白の指定（margin, padding）に不要な絶対値指定や複雑な指定がないか（中央寄せなどでメンテナンス性を損なう書き方をしていないか）
4. letter-spacing, line-height に適切な単位（em, unitlessなど）が使用されているか（px指定で崩れの原因になっていないか）
5. 禁止されている単位（例: 全てを vw で指定するなど、案件の禁止事項に抵触していないか）が使用されていないか
6. 英字の大文字表記が必要な箇所で、HTMLに直接大文字を書かず CSS の `text-transform: uppercase` を使用しているか
```

## レスポンシブ対応と表示崩れの防止

```
レスポンシブ実装とレイアウトについて確認してください。

1. 320px〜の最小幅からPC幅まで、レイアウト崩れや意図しない横スクロールが発生しそうな箇所がないか（CSSのwidth指定などを確認）
2. Google Maps や YouTube などの iframe がレスポンシブ対応されているか（画面幅からはみ出していないか）
3. ハンバーガーメニュー展開時に、背面のコンテンツスクロールが固定（ロック）される仕様になっているか（bodyへのoverflow: hidden付与など）
```

## インタラクションとアニメーション

```
ユーザー操作に対する反応について確認してください。

1. クリック可能なすべての要素（リンク、ボタン）に `:hover` スタイルが設定されているか
2. タッチデバイス（スマホ・タブレット）では hover アニメーションが無効化（または `@media (hover: hover)` で制御）されているか
3. ページ内リンクに対してスムーズスクロール（`scroll-behavior: smooth;` やJS制御）が適用されているか
4. アニメーションが遅延しすぎていたり、ユーザー体験を損なう動きになっていないか
```
