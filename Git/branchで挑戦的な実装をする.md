# 🚀 チャレンジ実装・開発マニュアル

「既存のコードを壊さず、新しい機能やAI実装を試したい」時のための標準ワークフローです。

## 1. 準備：現在の状態を安全に保存
作業を始める前に、現在の安定したコード（master/main）を最新の状態にします。

```bash
# masterブランチに移動
git checkout master

# リモートの最新状態を取り込む
git pull origin master
```

## 2. 出陣：新しい実験用ブランチを作成
メインのコードに影響を与えない専用の作業スペースを作ります。

```bash
# ブランチ作成と切り替えを同時に行う
# 名前は feature/機能名 などにすると分かりやすいです
git checkout -b feature/your-challenge-name
```
*   **確認方法:** `git branch` を実行し、作成したブランチ名に `*` が付いていれば成功です。

## 3. 実装：実験とこまめな記録
AIロジックやスクロール演出などのコードを書きます。**「ここまで動いた」というタイミングでこまめに保存**するのがコツです。

```bash
# 変更したファイルをステージング
git add .

# メッセージを添えて保存
git commit -m "AIのレスポンス処理の初期案を実装"
```

---

## 4. 終結：結果に応じた処理

実験が終わり、動作確認ができたら以下のいずれかを選択します。

### ✅ パターンA：成功！本番に取り込む（マージ）
```bash
# 1. 本番用ブランチに戻る
git checkout master

# 2. 実験内容を合流させる
git merge feature/your-challenge-name

# 3. サーバー（GitHub等）に反映する
git push origin master

# 4. (任意) 使い終わったブランチを削除
git branch -d feature/your-challenge-name
```

### ❌ パターンB：失敗…なかったことにする（削除）
「この方針はダメだ」となったら、ブランチごと捨てればメインのコードは無傷です。

```bash
# 1. 本番用ブランチに戻る
git checkout master

# 2. 失敗したブランチを強制削除（-Dは大文字）
git branch -D feature/your-challenge-name
```

## 💡 AI実装・開発のヒント
*   **パラメータの試行錯誤:** AIの温度感（Temperature）やプロンプトの調整ごとにコミットしておくと、「3つ前の設定が一番賢かった」という時にすぐ戻せます。
*   **リセットの呪文:** もしブランチ内で作業がめちゃくちゃになったら、`git reset --hard HEAD` を打てば、そのブランチの最後のコミット時点まで一気に巻き戻せます。

**「失敗してもブランチを消せばいいだけ」**。この安心感を持って、どんどんチャレンジしましょう！
